Cancel an activation whose status is PENDING. A successful cancellation results in a 200 response and an ABORTED status. The client can still poll the canceled activation, but it no longer appears in the list of activations. If the activation is no longer PENDING, a 422 (unprocessable) error indicates that it can't be canceled. Canceling an activation that has already been canceled results in a 204 response, indicating there's no resource to delete. Attempting to cancel an unknown activation results in a 404 error.
